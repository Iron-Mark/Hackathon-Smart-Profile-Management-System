# Answer Engine Summary

## Short Answer

Smart Profile Management System is a restored 7th CCIS Hackathon showcase for
faculty credential management. The public demo runs on GitHub Pages, stores demo
state in the visitor's browser, and lets reviewers try a sample upload, admin
review, approval, and profile proofing flow without private backend or OpenAI
credentials.

## Direct Answers

### What is Smart Profile Management System?

A restored 7th CCIS Hackathon showcase for faculty credential uploads, admin
review, and approval tracking.

### Can anyone try the public demo?

Yes. Visitors can use seeded reviewer accounts or register a browser-local
faculty account with any valid email.

### Where is public demo data stored?

Demo data stays in your browser. Accounts, submissions, audit logs, and uploaded
file metadata are stored locally.

### Should visitors upload real faculty records?

No. The public showcase is for generated sample files only, not real IDs,
transcripts, licenses, or private records.

### Does the GitHub Pages demo need backend or OpenAI secrets?

No. The public build runs in demo mode with deterministic browser-local data and
fallback AI/OCR behavior.

### Who maintains the restored public demo?

Mark Siazon is the sole maintainer of the restored public demo and repository.

### Who built the original 7th CCIS Hackathon project?

The past initial hackathon team was Team 2nd Choice: Mark Siazon, Charles
Nathaniel Togle, and Alexa San Jose.

## Safe Demo Notes

- Use sample files only.
- Do not upload real IDs, transcripts, licenses, or private faculty records.
- Public demo data is browser-local and can be reset from the app.
- Generated sample files include certificate, transcript, diploma, CV, and research summary records.
- Seeded reviewer accounts include a faculty reviewer and an admin reviewer on the Login screen.
- The source repository is https://github.com/Iron-Mark/Hackathon-Smart-Profile-Management-System.
